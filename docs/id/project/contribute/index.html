Commit ke repo all-contributors

Kami menggunakan kaidah commit angular untuk commit yang digunakan untuk pembuatan changelog otomatis, jadi disarankan untuk menghargai itu. Jika kamu tidak biasa atau takut salah mengetik commit, kamu bisa menjalankan npm run commit atau git cz (jika kamu sudah memasang commitizen) yang akan membawa kamu melalui beberapa pertanyaan dan menulis commit untuk kamu.

Terjemahan

Terjemahan kami dikelola melalui crowdin. Kamu dapat membantu berkontribusi dengan menuju the crowdin project dan menyarankan terjemahan. Jika kamu suka, komen di isu ini agar kami tahu bahwa kamu ingin membantu atau punya pertanyaan!

Saat menerjemahkan, penting bahwa yang memiliki XPATH @href, @src dll tidak boleh diterjemahkan. Hal-hal seperti code seharusnya juga tidak. Karena akan menyebabkan proyek menjadi 404. Lihat gambar dibawah sebagai contoh:

Namun, disarankan untuk menerjemahkan konten apa pun yang akan berkontribusi pada pemahaman yang lebih baik seperti komentar di tag code selama tidak mengubah kode secara aktual karena ngoding kan pake bahasa Inggris.

Bagaimana caranya

Masuk ke Crowdin

  1. Pergi ke proyek crowdin, lalu daftar terus masuk.
  2. Gabung ke proyek "All Contributors" dan jeng-jeng!

Menerjemahkan kalimat

  1. Pergi ke proyek crowdin.
  2. Klik bendera dari bahasa yang ingin kamu terjemahkan dokumentasinya.
  3. Klik pada setiap berkas *.md yang belum sepenuhnya diterjemahkan (lihat yang menunjukan 100% di bagian kanan)
  4. Tulis terjemahan kamu di kolom "Enter translation here" (yang akan sesuai dengan terjemahan teks dalam "SOURCE STRING" dikolom atas) lalu klik "SAVE"
  5. Jika seseorang (termasuk diri Anda) mengirimkan saran (yang akan muncul di "[LANGUAGE] TRANSLATIONS" pada bagian bawah) dan kamu yakin itu benar maka klik ikon :ballot_box_with_check: untuk menyetujuinya.

Ganti deployement

Setelah kamu menerjemahkan kalimat pada bahasa apapun, kamu tidak akan lihat langsung perubahannya sampai branch master sudah diperbarui diikuti oleh deployement yang berhasil.
